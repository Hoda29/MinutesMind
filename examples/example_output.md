## Notes on Output Quality

- **Short transcripts** (~500 words): Fast processing (~4s), high fidelity extraction, all action items captured
- **Medium transcripts** (~2000 words): Good processing time (~8s), nuanced summaries, complex multi-party action items correctly attributed
- **Long transcripts** (~5000+ words): Automatic chunking kicks in, each section summarized then synthesized; adds ~5–10s per extra chunk
- **Edge cases**: Transcripts with no clear action items return an empty array (not an error); vague assignees are labeled "Team" or "TBD"
